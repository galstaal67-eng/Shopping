# חוּט וָרַפְיָה — אתר בוטיק לתיקי סריגה בעבודת יד

אתר סטטי (HTML/CSS/JS, ללא build step) המציג קטלוג תיקי סריגה (חוט רפיה)
לפי קטגוריות, עם עמוד/חלון מוצר שכולל **צפייה מזוויות שונות** לכל תיק
(דרג עם העכבר/אצבע כדי לעבור בין התמונות שהועלו). נפרס אוטומטית כחלק
מהאתר הקיים (Netlify מפרסם את כל תיקיית הריפו), בנתיב:

```
/bags-boutique/
```

## מבנה התיקייה

```
bags-boutique/
  index.html          העמוד הראשי (הירו, קטגוריות, קטלוג, הסיפור שלנו, יצירת קשר)
  css/style.css        כל העיצוב
  js/products.js        נתוני המוצרים והקטגוריות
  js/spin-viewer.js      קומפוננטת צפייה רב-זוויתית לשימוש חוזר (עובדת עם 1+ תמונות)
  js/app.js             ניתוב/רינדור/אינטראקציות (tilt, פרלקסה, מודאל מוצר)
  images/products/<slug>/   תמונות המוצר (קובץ ראשון = תמונת קטלוג)
  images/products/_unsorted/ תמונות שהועלו ולא שויכו אוטומטית לתיק מסוים
  tools/generate_placeholder_bags.py  סקריפט ליצירת איורי-ממלא (SVG) למוצר חדש שעדיין אין לו תמונות
```

## ⚠️ מיון אוטומטי של התמונות — כדאי לבדוק

32 התמונות שהועלו הגיעו ללא מיון (קובצי `IMG-...-WA00xx.jpg` בתיקייה
אחת). הן קובצו ל-17 מוצרים לפי זיהוי חזותי (צבע/סגנון) — **ניחוש
ממוחשב, לא מדויק במאה אחוז**. יש לעבור על `js/products.js` ולוודא:

- שהתמונות בכל מוצר (`files`) אכן שייכות לאותו תיק בפועל.
- שהשמות (`name`), התיאורים (`description`) והמחירים (`price`) — שהם
  כרגע ניחושים סבירים בלבד — מעודכנים לפרטים האמיתיים.
- התיקייה `images/products/_unsorted/` מכילה כמה תמונות קבוצתיות/לא
  ברורות שלא שויכו לשום מוצר — אפשר לשבץ אותן ידנית או למחוק.

## איך מוסיפים מוצר חדש עם תמונות אמיתיות

לכל מוצר יש slug (מזהה) שמוגדר ב-`js/products.js`, למשל `pouch-cognac`.

1. שימו את התמונות של המוצר (JPG/PNG/WebP) בתיקייה
   `images/products/<slug>/` — כל תמונה היא "זווית" אחת. תמונה אחת
   עובדת מצוין (המוצר יוצג כתצלום רגיל); כמה תמונות (2 ומעלה) מאפשרות
   למבקר לגרור ולעבור ביניהן; 8+ תמונות ברצף סיבובי אמיתי (turntable)
   ייתנו חוויית "סיבוב 360°" מלאה והתגית תתעדכן אוטומטית.
2. הוסיפו/עדכנו את אובייקט המוצר ב-`js/products.js` — שדה `files`
   הוא רשימת שמות הקבצים בתיקייה, לפי סדר התצוגה הרצוי.

צפיין הזוויות (`js/spin-viewer.js`) עובד עם **כל מערך תמונות**, מ-1
ומעלה — אין צורך לשנות קוד.

### אם עדיין אין תמונות למוצר

`tools/generate_placeholder_bags.py` (Python 3, ללא תלויות חיצוניות)
מייצר איור וקטורי זמני של תיק (כיסוי + 12 פריימי סיבוב) בתיקיית
`images/products/<slug>/`, כדי שהקטלוג יישאר שלם עד שיעלו תמונות אמיתיות.

## הוספת/עריכת מוצרים

כל מוצר הוא אובייקט ב-`PRODUCTS` בתוך `js/products.js`:

```js
{
  slug: "pouch-cognac",       // מזהה ייחודי, גם שם התיקייה תחת images/products/
  name: "קלמית ‘קוניאק’",
  category: "pouch",          // חייב להתאים ל-id באחת מ-CATEGORIES
  price: 130,
  material: "סריגת יד בחוט רפיה בגוון חום קוניאק",
  badge: "פריט יחיד",
  description: "...",
  files: ["IMG-20260718-WA0014.jpg"],
}
```

קטגוריות חדשות מוגדרות במערך `CATEGORIES` באותו קובץ (id, name, tagline, icon).

## פרטי יצירת קשר

בראש `js/app.js` יש אובייקט `CONTACT` עם מספר וואטסאפ (בפורמט בינלאומי,
בלי `+` או `0` מוביל) וכתובת מייל — יש לעדכן לפני עלייה לאוויר:

```js
const CONTACT = {
  whatsappNumber: "972500000000",
  whatsappMessage: "היי! מתעניין/ת בתיק מהאתר",
  email: "hello@example.com",
};
```

## הרצה מקומית

אין צורך ב-build. פשוט הריצו שרת סטטי מתיקיית `bags-boutique`, למשל:

```bash
python3 -m http.server 8080
```

ופתחו `http://localhost:8080/`.
