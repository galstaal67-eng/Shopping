# עוֹר וָחוּט — אתר בוטיק לתיקי עור בעבודת יד

אתר סטטי (HTML/CSS/JS, ללא build step) המציג קטלוג תיקים לפי קטגוריות,
עם עמוד/חלון מוצר שכולל **צפייה בסיבוב 360°** לכל תיק. נפרס אוטומטית
כחלק מהאתר הקיים (Netlify מפרסם את כל תיקיית הריפו), בנתיב:

```
/bags-boutique/
```

## מבנה התיקייה

```
bags-boutique/
  index.html          העמוד הראשי (הירו, קטגוריות, קטלוג, הסיפור שלנו, יצירת קשר)
  css/style.css        כל העיצוב
  js/products.js        נתוני המוצרים והקטגוריות
  js/spin-viewer.js      קומפוננטת צפיית 360° לשימוש חוזר
  js/app.js             ניתוב/רינדור/אינטראקציות (ניתן ל-tilt, פרלקסה, מודאל מוצר)
  images/products/<slug>/cover.svg        תמונת קטלוג לכל מוצר
  images/products/<slug>/360/frame-NN.svg  12 פריימים לסיבוב 360°
  tools/generate_placeholder_bags.py       הסקריפט שיצר את איורי הדוגמה
```

כרגע כל התמונות הן **איורים וקטוריים זמניים** שנוצרו אוטומטית (SVG) —
כדי שהאתר יהיה שלם ופונקציונלי כבר עכשיו. יש להחליף אותן בתמונות אמיתיות.

## איך להעלות תמונות אמיתיות

לכל מוצר יש slug (מזהה) שמוגדר ב-`js/products.js`, למשל `tote-caramel`.
כדי להחליף את התמונות שלו בתמונות אמיתיות:

1. **תמונת קטלוג** — שימו תמונה (JPG/PNG/WebP, יחס גובה-רוחב מומלץ ~4:5)
   בנתיב `images/products/tote-caramel/cover.jpg` והחליפו את ההפניה
   ל-`cover` באובייקט המוצר המתאים ב-`js/products.js` (או שמרו את אותו
   שם קובץ `cover.svg`/`cover.jpg` ועדכנו את הקוד ב-`products.js` בהתאם).

2. **סדרת 360°** — צלמו את התיק על חצובה מסתובבת (turntable) או סובבו
   אותו ידנית במרווחים שווים, ושמרו **12–36 תמונות** (ככל שיותר — הסיבוב
   חלק יותר) בתיקייה `images/products/tote-caramel/360/`, בשמות
   רציפים כמו `frame-00.jpg`, `frame-01.jpg` וכו'. עדכנו את הפונקציה
   `framePaths()` ב-`js/products.js` להצביע לקבצי ה-JPG החדשים (ואת
   `FRAME_COUNT` אם השתמשתם במספר פריימים שונה מ-12).

3. מוחקים את קובצי ה-SVG הישנים של אותו מוצר (או משאירים אותם — הם לא
   ייטענו אם עדכנתם את הנתיב ב-`products.js`).

צפיין ה-360° (`js/spin-viewer.js`) עובד עם **כל מערך תמונות מסודר** —
אין צורך לשנות שם קוד, רק להצביע למקור הנכון.

## הוספת/עריכת מוצרים

כל מוצר הוא אובייקט ב-`PRODUCTS` בתוך `js/products.js`:

```js
{
  slug: "tote-caramel",      // מזהה ייחודי, גם שם התיקייה תחת images/products/
  name: "טוט ‘כרמל’",
  category: "tote",          // חייב להתאים ל-id באחת מ-CATEGORIES
  price: 420,
  material: "עור פרה טבעי, עיבוד בגוון קרמל",
  badge: "עבודת יד",
  description: "...",
}
```

קטגוריות חדשות מוגדרות במערך `CATEGORIES` באותו קובץ (id, name, tagline, icon).

## פרטי יצירת קשר

בראש `js/app.js` יש אובייקט `CONTACT` עם מספר וואטסאפ (בפורמט בינלאומי,
בלי `+` או `0` מוביל) וכתובת מייל — יש לעדכן לפני עלייה לאוויר:

```js
const CONTACT = {
  whatsappNumber: "972500000000",
  whatsappMessage: "היי! מתעניין/ת בתיק מהאתר",
  email: "hello@example.com",
};
```

## הרצה מקומית

אין צורך ב-build. פשוט הריצו שרת סטטי מתיקיית `bags-boutique`, למשל:

```bash
python3 -m http.server 8080
```

ופתחו `http://localhost:8080/`.
