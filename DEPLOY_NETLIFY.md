# פריסת הדשבורד ל-Netlify

הדשבורד הוא קובץ HTML יחיד. במצב מתארח הוא עובד ב**מצב אחסון-דפדפן (IndexedDB)**:
האתר נפתח, טוען אוטומטית את `climate_data.xlsx` המצורף (אם קיים) בפעם הראשונה,
ומשם **שומר וטוען מ-IndexedDB של הדפדפן**. כפתור **⬇️ הורד Excel** מוריד עותק מעודכן בכל עת.

> חשוב להבין: אתר סטטי **לא כותב חזרה** לקובץ שבשרת. כל העריכות נשמרות בדפדפן
> שלך (לכל דפדפן/מכשיר בנפרד). כדי לקבל קובץ Excel מעודכן — לחץ **⬇️ הורד Excel**.
> כדי לשתף נתונים בין מכשירים/משתמשים נדרש backend (ראו "המשך אפשרי" בתחתית).

## מבנה החבילה

```
site/
  index.html          ← הדשבורד (עותק של climate_dashboard.html)
  climate_data.xlsx   ← קובץ הנתונים שלך (אופציונלי, מומלץ מדולל)
netlify.toml          ← מפרסם את site/
```

## שלב 1 — הכנת קובץ הנתונים (מדולל)

`climate_data.xlsx` המקורי גדול (~60MB, בגלל גיליונות המפה נקודות/Streams). לטעינה מהירה:

1. פתחו את `climate_dashboard.html` מקומית ב-**Chrome/Edge**.
2. **📂 חבר קובץ Excel** → בחרו את ה-xlsx המלא שלכם (נטען כרגיל).
3. לחצו **🌐 קובץ לאתר** → יורד `climate_data.xlsx` מדולל (גיליונות המפה מדוללים לתקרת שורות; שאר הנתונים נשמרים במלואם).
4. העתיקו את הקובץ שהורד אל `site/` (ליד `index.html`).

(אם לא תוסיפו קובץ — האתר עובד על נתוני ברירת המחדל המוטמעים בקוד.)

## שלב 2 — פריסה

### אפשרות א׳: גרירה ושחרור (הכי מהיר)
1. היכנסו ל-https://app.netlify.com → **Add new site → Deploy manually**.
2. גררו את תיקיית **`site/`** כולה לאזור ההעלאה.
3. האתר עולה לכתובת `https://<name>.netlify.app`.

### אפשרות ב׳: פריסה מ-Git
1. דחפו את המאגר ל-GitHub/GitLab.
2. ב-Netlify: **Add new site → Import from Git** → בחרו את הריפו.
3. `netlify.toml` כבר מגדיר `publish = "site"` — אין צורך בפקודת build.
4. כל push מעדכן את האתר.

### אפשרות ג׳: Netlify CLI
```bash
npm i -g netlify-cli
netlify deploy --dir=site --prod
```

## דרישות דפדפן
- כל דפדפן מודרני (Chrome/Edge/Firefox/Safari) — מצב אחסון-דפדפן (IndexedDB) + ⬇️ הורד Excel.
- **📂 חבר קובץ Excel** (שמירה ישירה לקובץ מקומי) — רק Chrome/Edge (File System Access API).
- האתר נטען מ-CDN (Leaflet/SheetJS) וטאב ההצפות מושך נתונים חיים מ-ArcGIS — נדרש אינטרנט.

## עדכון נתונים אחרי פריסה
- עבדו על האתר → השינויים נשמרים ב-IndexedDB.
- **⬇️ הורד Excel** מתי שתרצו עותק מעודכן.
- לרענון קובץ הבסיס של האתר עצמו: ייצרו `climate_data.xlsx` חדש (שלב 1) ופרסו מחדש.
  כדי שביקור יטען את הבסיס החדש (ולא את ה-IndexedDB הישן), נקו את אחסון האתר בדפדפן
  (DevTools → Application → IndexedDB → climate_dashboard → מחיקה) ורעננו.

## המשך אפשרי (שיתוף בין מכשירים)
אם בעתיד תרצו שהנתונים יהיו משותפים ויישמרו בענן — אפשר להוסיף Netlify Functions +
אחסון (למשל Netlify Blobs / בסיס נתונים). זה משנה את מודל השמירה מ"דפדפן" ל"שרת".
